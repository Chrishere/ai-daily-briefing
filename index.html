来源：magazine.sebastianraschka.com · Using Local Coding Agents（6 月 27 日）
总结：Sebastian Raschka 撰文详细讲解如何用完全本地化的开源方案搭建编码智能体，替代 Claude Code、Codex 等订阅服务。他给出的理由包括：达到订阅额度上限后成本可预测、不受 API 涨价影响，以及离线可用（比如坐飞机或去没有 Starlink 的小屋写代码）。实测发现，Qwen3.6 35B-A3B 在 Mac Mini M4 上跑出约 40 token/秒，能力与高推理档位的 GPT-5.5 相当；在 5 项基准任务中，Qwen3.6 与 North Mini Code 1.0 都完成了 4 项，而 Gemma 4 E2B 大多失败；此外 Codex 消耗的 token 最少，Claude Code 消耗的输入 token 约为其两倍。

Predictable, fixed costs if you reach your subscription plan limits, and immunity to API price changes. Reproducibility; sometimes it's nice if a model is upgraded (e.g., GPT 5.4 -> GPT 5.5 -> GPT 5.6) and it solves all your queries more reliably. However, this can also break existing workflows. Offline use in the classic airplane flight scenario with slow or no internet, or when going on a coding/writing retreat in the cabin in the woods w/o a Starlink subscription.
